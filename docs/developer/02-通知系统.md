# 通知系统

> 对应思维导图节点：通知系统 → 用户接收（读取/删除）、管理员管理（发送/撤销发送）

## 1. 功能概述

通知系统分为两个角色维度：

```
通知系统
├── 用户接收
│   ├── 读取（标记已读/全部已读）
│   └── 删除（单条删除/清空全部）
└── 管理员管理
    ├── 发送（单用户/批量/全体）
    └── 撤销发送（软删除，保留记录）
```

## 2. 数据库表结构

```sql
CREATE TABLE notifications (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    title TEXT NOT NULL,
    content TEXT NOT NULL,
    type TEXT DEFAULT 'system',
    is_read INTEGER DEFAULT 0,
    is_recalled INTEGER DEFAULT 0,       -- 管理员撤销标记（软删除）
    created_at TEXT DEFAULT (datetime('now')),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

## 3. 用户端 API

### 3.1 读取通知

```
GET /api/notifications?page=1&page_size=20&unread_only=false
Response: {
    data: [{ id, userId, title, content, type, isRead, isRecalled, createdAt }],
    total: number,
    page: number,
    pageSize: number
}
```

- 分页加载，默认每页 20 条
- `unread_only=true` 仅显示未读通知
- 自动过滤 `is_recalled=1` 的记录
- 按 `created_at` 降序排列

### 3.2 未读计数

```
GET /api/notifications/unread-count
Response: { data: { count: number } }
```

前端 `NotificationBell` 组件轮询此接口（每 30 秒），在导航栏显示未读红点。

### 3.3 标记已读

```
PUT /api/notifications/{id}/read         # 单条标记
PUT /api/notifications/read-all          # 全部标记
```

### 3.4 删除通知

```
DELETE /api/notifications/{id}           # 单条删除
DELETE /api/notifications/clear-all      # 清空全部
```

## 4. 管理员端 API

### 4.1 发送通知

**单用户发送：**
```
POST /api/notifications
Body: { user_id: number, title: string, content: string, type: string }
```

**批量发送：**
```
POST /api/notifications/batch
Body: { title: string, content: string, type: string, user_ids?: number[] }
```
- `user_ids` 为空时发送给全体用户

### 4.2 已发送列表

```
GET /api/notifications/sent?page=1&page_size=20
Response: {
    data: [{ id, userId, title, content, type, isRead, isRecalled, createdAt }],
    total: number,
    page: number,
    pageSize: number
}
```

### 4.3 撤销发送

管理员可软删除已发送的通知（`is_recalled=1`）：

```
PUT /api/notifications/{id}/recall       # 管理员撤销
```

> 撤销后通知对用户不可见，但数据库保留记录。

## 5. 通知类型

| type 值 | 说明 | 图标 | 颜色 |
|---------|------|------|------|
| `system` | 系统通知 | ⚙️ `fa-gear` | 灰色 |
| `workflow` | 工作流通知 | 🔀 `fa-diagram-project` | 蓝色 |
| `patent` | 专利通知 | 📄 `fa-file-lines` | 紫色 |
| `alert` | 告警通知 | ⚠️ `fa-triangle-exclamation` | 红色 |

## 6. 前端实现

### 6.1 NotificationBell（通知铃铛）

位置：`AppLayout` 顶栏右侧

功能：
- 显示未读通知数（红点 badge）
- 点击展开通知下拉列表
- 每 30 秒轮询未读计数
- 通知列表项显示类型图标
- 详情弹窗显示类型图标

### 6.2 相关文件

| 文件 | 说明 |
|------|------|
| `backend/app/tables/notifications.py` | 通知表定义（含 is_recalled） |
| `backend/app/api/notifications.py` | 通知路由（10 个端点） |
| `frontend/src/api/notifications.ts` | 前端通知 API 客户端 |
| `frontend/src/components/layout/AppLayout.tsx` | 通知铃铛 + 下拉列表 + 详情弹窗 |

## 7. API 端点汇总

| 端点 | 方法 | 角色 | 说明 |
|------|------|------|------|
| `/api/notifications` | GET | 用户 | 读取通知列表（分页） |
| `/api/notifications/unread-count` | GET | 用户 | 未读计数 |
| `/api/notifications/{id}/read` | PUT | 用户 | 标记单条已读 |
| `/api/notifications/read-all` | PUT | 用户 | 标记全部已读 |
| `/api/notifications/{id}` | DELETE | 用户 | 单条删除 |
| `/api/notifications/clear-all` | DELETE | 用户 | 清空全部 |
| `/api/notifications` | POST | 管理员 | 单用户发送 |
| `/api/notifications/batch` | POST | 管理员 | 批量/全体发送 |
| `/api/notifications/sent` | GET | 管理员 | 已发送列表 |
| `/api/notifications/{id}/recall` | PUT | 管理员 | 撤销发送（软删除） |
