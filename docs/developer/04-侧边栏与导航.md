# 侧边栏与导航

> 对应思维导图节点：侧边栏（普通用户公共模块、管理员附加）、侧边栏状态同步

## 1. 侧边栏结构

```
侧边栏
├── 普通用户（公共模块）
│   ├── 首页
│   ├── 知识库
│   ├── 历史方案
│   └── 专利转化
├── 工作流步骤（运行时显示）
│   ├── 需求画像
│   ├── 问题建模
│   ├── 专利检索
│   ├── 方案生成
│   └── 方案评估
└── 管理员附加（仅 admin 显示）
    ├── 模型服务
    ├── 用户管理
    └── 数据监控
```

## 2. 普通用户公共模块

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | `DashboardPage` | 首页仪表板 |
| `/knowledge` | `KnowledgeBasePage` | 知识库管理 |
| `/history` | `HistoryPage` | 历史方案浏览 |
| `/patent-conversion` | `PatentConversionPage` | 专利转化 |

## 3. 管理员附加模块

| 路径 | 组件 | 说明 |
|------|------|------|
| `/admin/keys` | `KeyManagementPage` | 模型服务（AI 供应商 + Key 池管理） |
| `/admin/users` | `UserManagementPage` | 用户管理（列表/角色/删除） |
| `/monitor` | `MonitorPage` | 数据监控面板（系统/Key/任务统计） |

管理员模块仅在 `role === 'admin'` 时显示在侧边栏底部「管理员」区域。

## 4. 侧边栏状态同步

### 4.1 需求

当工作流运行时，侧边栏实时显示各阶段进度：

```
┌─ 流程进度 ─────────────┐
│ ✓ 需求画像  [完成]      │
│ ● 问题建模  [进行中]    │
│ ○ 专利检索  [等待]      │
│ ○ 方案生成  [等待]      │
│ ○ 方案评估  [等待]      │
└────────────────────────┘
```

### 4.2 实现方案

**状态管理（Zustand Store）：**

```typescript
// store/useWorkflowStore.ts
interface WorkflowState {
  taskId: string | null;
  currentPhase: string | null;
  phaseStatus: Record<string, 'pending' | 'running' | 'completed' | 'failed'>;
  isRunning: boolean;
  startWorkflow: (taskId: string) => Promise<void>;
  pollStatus: () => void;
}
```

**后端状态端点：**

```
GET /api/workflow/{task_id}/status
Response: {
    currentPhase: 'problem_modeling',
    phaseStatus: {
        demand_portrait: 'completed',

        evaluation: 'pending'
    },
    startedAt: '2024-01-01T00:00:00Z',
    elapsed: 45.2
}
```
