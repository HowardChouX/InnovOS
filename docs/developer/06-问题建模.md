# 问题建模

> 对应思维导图节点：问题建模（资源分析、进化趋势分析、物-场分析、创新方向 → 用户交互评分）

## 1. 功能概述

问题建模阶段在需求洞察的基础上，进一步分析系统资源和进化方向，结合物-场模型，生成创新方向供用户评分。3 个分析器并行执行，结果汇总后由 AI 综合生成创新方向。

```
问题建模
├── 资源分析      → 物质/场/信息/功能/时间/空间/结构（复用需求洞察分析）
├── 进化趋势分析  → S 曲线 + TRIZ 8 大进化法则
├── 物-场分析     → Su-Field 模型（S1/S2/F）+ 问题类型判断
└── 创新方向生成  → AI 综合 3 个分析结果生成
    → 用户星星评分
```

## 2. 分析器详情

### 2.1 资源分析（ResourceAnalyzer）

**源文件**：`backend/app/algorithm/analyzers/resource_analyzer.py`

分析系统中可用的物质、场、信息、功能、时间、空间、结构七种资源。与需求洞察阶段的资源分析相同，结果可供后续阶段共用。

### 2.2 进化趋势分析（EvolutionAnalyzer）

**源文件**：`backend/app/algorithm/analyzers/evolution_analyzer.py`

基于 TRIZ 8 大进化法则分析系统的技术进化方向，输出当前 S 曲线阶段和进化趋势：

| 进化法则 | 说明 |
|---------|------|
| 向超系统进化 | 系统功能向更高层次整合 |
| 向微级进化 | 系统元件向微观发展 |
| 柔性化/动态化 | 系统向可调节/可移动方向进化 |
| 多功能性 | 系统功能数量增加 |
| 不均衡性 | 系统各部分发展不均衡 |
| 向上进化 | 系统理想度提高 |
| 由宏观到微观 | 物理结构向微观转变 |
| 协调性 | 系统各参数协调匹配 |

### 2.3 物-场分析（SuFieldAnalyzer）

**源文件**：`backend/app/algorithm/analyzers/sufield_analyzer.py`

构建系统的物-场模型（S1-S2-F），识别交互问题类型：
- **有害作用**（harmful）— 需要消除
- **不足作用**（insufficient）— 需要增强
- **过度作用**（excessive）— 需要削弱
- **缺失作用**（missing）— 需要建立

## 3. 创新方向生成

3 个分析器结果汇总后，由编排器调用 AI 综合生成创新方向列表：

```python
# algorithm/analyzers/problem_modeling.py
class ProblemModelingAnalyzer:
    """问题建模编排器"""

    async def analyze(self, problem: str, demand_results: dict | None = None) -> dict:
        # 并行执行 3 个分析器
        resource_result, evolution_result, sufield_result = await asyncio.gather(
            self.resource_analyzer.analyze(problem),
            self.evolution_analyzer.analyze(problem),
            self.sufield_analyzer.analyze(problem),
        )

        # AI 综合生成创新方向
        innovations = await self._generate_innovations(
            problem, resource_result, evolution_result, sufield_result, demand_results
        )

        return {
            "resource_analysis": resource_result,
            "evolution_trend": evolution_result,
            "sufield_analysis": sufield_result,
            "innovations": innovations,
        }
```

## 4. 用户交互评分

创新方向列表格式：

```json
{
    "innovations": [
        {
            "id": "in1",
            "source": "进化趋势分析",
            "description": "采用模块化设计实现系统动态化",
            "principle": "柔性化/动态化法则",
            "user_rating": null
        }
    ]
}
```

用户对每个创新方向进行星星评分，评分提交后触发 `MODELING_DONE` 事件进入下一步。

## 5. 数据库表

```sql
CREATE TABLE problem_modelings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id INTEGER UNIQUE NOT NULL,
    resource_analysis JSON,
    evolution_trend JSON,
    sufield_analysis JSON,
    innovations JSON,
    created_at TEXT DEFAULT (datetime('now')),
    FOREIGN KEY (task_id) REFERENCES tasks(id)
);
```
