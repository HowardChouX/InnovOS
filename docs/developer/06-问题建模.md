# 问题建模

> 对应思维导图节点：问题建模（功能分析、因果链分析、矛盾分析、裁剪分析、进化趋势分析、创新方向 → 用户交互评分）

## 1. 功能概述

问题建模阶段在需求画像的基础上，深入分析系统内部结构和矛盾，生成创新方向供用户评分。

```
问题建模
├── 功能分析（RootSeek）→ 组件识别、交互分析、问题类型判断
├── 因果链分析（RootSeek）→ 根因追溯
├── 矛盾分析（RootSeek）→ 技术矛盾/物理矛盾
├── 裁剪分析（RootSeek）→ 组件精简方案
├── 进化趋势分析（RootSeek）→ 技术系统进化方向
└── 创新方向 → 根据建模数据生成创新方向列表
    → 用户对创新方向交互星星评分
```

## 2. 分析器详情（从 RootSeek 迁移）

### 2.1 功能分析（FunctionAnalyzer）

**RootSeek 源文件**：`src/ai/analyzers/function_analyzer.py`
**InnovOS 目标**：`algorithm/analyzers/function_analyzer.py`

功能分析分 3 步执行：

1. **系统定义 + 超系统组件识别**：判断系统类型（硬件/软件/流程/服务），识别外部超系统组件
2. **系统组件枚举**：枚举系统内部组件（排除超系统）
3. **扩展补全**：子组件拆分 + 遗漏检查

然后进行**逐对交互分析**：判断每对组件的接触关系和交互类型（harmful/useful/insufficient/excessive）。

**适配要点**：
- RootSeek 的 `FunctionAnalyzer` 使用 `self.ai.call_ai()`，InnovOS 需要注入 `AIBase` 实例
- 提示词模板需迁移到 `algorithm/prompts/problem_modeling/function_analysis.py`

### 2.2 因果链分析（RootCauseAnalyzer）

**RootSeek 源文件**：`src/ai/analyzers/root_cause_analyzer.py`

从表面问题出发，逐层追溯因果关系，找到根本原因。

输出结构：
```json
{
    "causal_chain": [
        {"effect": "表面问题", "cause": "直接原因", "level": 0},
        {"effect": "直接原因", "cause": "深层原因1", "level": 1},
        {"effect": "深层原因1", "cause": "根本原因", "level": 2}
    ],
    "root_cause": "根本原因描述"
}
```

### 2.3 矛盾分析

基于功能分析的交互结果，识别技术矛盾和物理矛盾：

- **技术矛盾**：改善参数 A 导致参数 B 恶化
- **物理矛盾**：同一参数需要满足相反需求

输出：矛盾矩阵对（推荐 40 个发明原理中的对应原理）。

### 2.4 裁剪分析（TrimmingAnalyzer）

**RootSeek 源文件**：`src/ai/analyzers/trimming_analyzer.py`

分析系统中可裁剪的组件，提出精简方案：
- 哪些组件可以删除
- 删除后功能如何由其他组件或超系统承担

### 2.5 进化趋势分析（EvolutionAnalyzer）

**RootSeek 源文件**：`src/ai/analyzers/evolution_analyzer.py`

基于 TRIZ 8 大进化法则分析系统的技术进化方向：
1. 向超系统进化
2. 向微级进化
3. 柔性化/动态化
4. 多功能性
5. 不均衡性
6. 向上进化
7. 由宏观到微观
8. 协调性

## 3. 创新方向生成

所有分析器的结果汇总后，由编排器生成创新方向列表：

```python
# algorithm/analyzers/problem_modeling.py
class ProblemModelingAnalyzer:
    """问题建模编排器。"""

    async def analyze(self, problem: str, demand_results: dict) -> dict:
        # 并行执行各分析器
        function_result = await self.function_analyzer.analyze(problem)
        causal_result = await self.root_cause.analyze(problem)
        contradiction_result = await self.contradiction_analyzer.analyze(problem, function_result)
        trimming_result = await self.trimming_analyzer.analyze(problem, function_result)
        evolution_result = await self.evolution_analyzer.analyze(problem)

        # 生成创新方向
        innovations = await self._generate_innovation_directions(
            function_result, causal_result, contradiction_result,
            trimming_result, evolution_result, demand_results
        )

        return {
            "function_analysis": function_result,
            "causal_chain": causal_result,
            "contradictions": contradiction_result,
            "trimming": trimming_result,
            "evolution": evolution_result,
            "innovation_directions": innovations  # 创新方向列表
        }
```

## 4. 用户交互评分

创新方向列表与需求画像的评分机制一致：

```json
{
    "innovations": [
        {
            "id": "in1",
            "source": "矛盾分析",
            "description": "采用可变刚度材料解决强度-重量矛盾",
            "principle": "发明原理 #35：参数变化",
            "user_rating": null
        }
    ]
}
```

## 5. 数据库表

```sql
CREATE TABLE problem_modelings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id INTEGER UNIQUE NOT NULL,
    function_analysis JSON,
    causal_chain JSON,
    contradictions JSON,
    trimming JSON,
    evolution JSON,
    innovations JSON,
    created_at TEXT DEFAULT (datetime('now')),
    FOREIGN KEY (task_id) REFERENCES tasks(id)
);
```
