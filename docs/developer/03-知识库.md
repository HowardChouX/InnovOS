# 知识库

> 对应思维导图节点：知识库（借鉴 CherryStudio 接入方式）

## 1. 功能概述

知识库模块允许用户管理个人知识文档，并通过 RAG（检索增强生成）为 AI 工作流提供上下文参考。当前实现为文本存储+LIKE 搜索，已通过 `algorithm/knowledge/` 模块实现向量检索能力。

## 2. 当前实现状态

### 2.1 已实现（✅）

| 功能 | 说明 |
|------|------|
| 文档 CRUD | 创建/读取/更新/删除知识库文档 |
| 分类管理 | 按 category 分组统计 |
| 文本搜索 | `LIKE '%keyword%'` 降级搜索 |
| 文档分块 | `chunker.py` 递归段落+句子分块，支持重叠窗口 |
| 向量存储 | `vector_store.py` numpy 余弦相似度检索（numpy 不可用时纯 Python 降级） |
| 向量嵌入 | `embedder.py` 支持三种模式：hash 降级、远程 API、本地 sentence-transformers |
| RAG 检索管线 | `retriever.py` 分块 → 向量化 → 存储 → 检索 |
| 自动索引 | 创建/更新文档时自动索引到向量存储 |
| 分块数据库表 | `knowledge_chunks` 表持久化分块数据 |

### 2.2 CherryStudio 参考架构

```text
CherryStudio 知识库管线：
1. 文档读取（readers/）→ 支持 PDF/DOCX/EPUB/TXT/MD/URL
2. 文档分块（utils/indexing/chunk.ts）→ 按段落+语义分块
3. 向量嵌入（utils/indexing/embed.ts）→ 调用 Embedding API
4. 向量存储（vectorstore/）→ LibSQL 向量存储
5. 相似度检索（utils/search.ts）→ 余弦相似度 Top-K
6. 可选 Rerank（rerank/）→ 重排序提升精度
```

InnovOS 适配差异：
- **向量存储**：使用 numpy + 余弦相似度（MVP），比 LibSQL 更轻量
- **嵌入模型**：通过已有的 Provider 系统配置 Embedding API Key（如 SiliconFlow 的 BGE 系列）
- **分块策略**：递归按段落→句子→字符，与 CherryStudio 一致

## 4. API 端点

现有知识库端点保持兼容，`/api/knowledge/search` 已升级为向量检索优先：

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/knowledge/docs` | GET | 文档列表（分页+分类+搜索） |

| `/api/knowledge/docs/{id}` | GET/PUT/DELETE | 文档详情/更新/删除 |
| `/api/knowledge/categories` | GET | 分类统计 |
| `/api/knowledge/search` | GET | 检索（LIKE 降级 + 向量检索） |

CREATE TABLE knowledge_chunks (
