# 工作流状态机

> 对应思维导图节点：流程步骤模块

## 1. 状态机设计

### 1.1 阶段定义

参考 RootSeek 的 `PhaseStateMachine`（`src/core/state_machine.py`），InnovOS 使用 6 阶段状态机：

```python
# algorithm/workflow_state_machine.py
from enum import Enum

class WorkflowPhase(Enum):
    DEMAND_PORTRAIT = "demand_portrait"       # 需求画像
    PROBLEM_MODELING = "problem_modeling"     # 问题建模
    PATENT_SEARCH = "patent_search"           # 专利检索
    SOLUTION_GEN = "solution_gen"             # 方案生成
    EVALUATION = "evaluation"                 # 方案评估
    COMPLETED = "completed"                   # 完成

    @property
    def label(self) -> str:
        return {
            "demand_portrait": "需求画像",
            "problem_modeling": "问题建模",
            "patent_search": "专利检索",
            "solution_gen": "方案生成",
            "evaluation": "方案评估",
            "completed": "完成",
        }[self.value]

class WorkflowEvent(Enum):
    DEMAND_DONE = "demand_done"
    MODELING_DONE = "modeling_done"
    PATENT_DONE = "patent_done"
    SOLUTION_DONE = "solution_done"
    EVAL_DONE = "eval_done"
    RETRY_MODELING = "retry_modeling"       # 评估后重新建模
    RETRY_SOLUTION = "retry_solution"       # 评估后调整方案
```

### 1.2 状态转移表

```python
_TRANSITIONS = {
    (WorkflowPhase.DEMAND_PORTRAIT, WorkflowEvent.DEMAND_DONE):
        WorkflowPhase.PROBLEM_MODELING,
    (WorkflowPhase.PROBLEM_MODELING, WorkflowEvent.MODELING_DONE):
        WorkflowPhase.PATENT_SEARCH,
    (WorkflowPhase.PATENT_SEARCH, WorkflowEvent.PATENT_DONE):
        WorkflowPhase.SOLUTION_GEN,
    (WorkflowPhase.SOLUTION_GEN, WorkflowEvent.SOLUTION_DONE):
        WorkflowPhase.EVALUATION,
    (WorkflowPhase.EVALUATION, WorkflowEvent.EVAL_DONE):
        WorkflowPhase.COMPLETED,
    # 循环路径
    (WorkflowPhase.EVALUATION, WorkflowEvent.RETRY_MODELING):
        WorkflowPhase.PROBLEM_MODELING,
    (WorkflowPhase.EVALUATION, WorkflowEvent.RETRY_SOLUTION):
        WorkflowPhase.SOLUTION_GEN,
}
```

### 1.3 状态机实现

从 RootSeek 的 `PhaseStateMachine` 移植，主要变更：

| RootSeek | InnovOS |
|----------|---------|
| `TRIZPhase`（5 阶段） | `WorkflowPhase`（6 阶段） |
| 手动事件触发 | FastAPI 后台任务自动触发 |
| 单用户 | 多用户并发（每任务独立状态机） |
| 内存状态 | 持久化到 `workflows` 表 |

## 2. 前端 Agent 定义

前端 `types/workflow.ts` 定义了 6 Agent 管线（与后端 5 Phase + COMPLETED 一一对应）：

| 序号 | Agent | 后端 Phase | 说明 |
|------|-------|-----------|------|
| ① | 需求洞察Agent | `DEMAND_PORTRAIT` | 理解用户需求，提取关键要素 |
| ② | 问题建模Agent | `PROBLEM_MODELING` | 构建问题模型，识别核心冲突 |
| ③ | 专利分析Agent | `PATENT_SEARCH` | 检索相关专利，分析技术方案 |
| ④ | 方案生成Agent | `SOLUTION_GEN` | 生成创新方案，整合多源知识 |
| ⑤ | 方案评估Agent | `EVALUATION` | 评估方案可行性与创新性 |
| ⑥ | 成果转化Agent | `COMPLETED`（完成后展示） | 输出结构化成果，支持转化 |

> 成果转化 Agent 是前端展示层概念——当后端 EVALUATION 阶段完成并触发 COMPLETED 后，前端自动显示成果转化面板，汇总全部阶段数据并输出结构化成果。

## 3. 工作流编排

### 3.1 执行流程

```
用户点击「开始分析」
  ↓
创建 workflow 记录（status=pending）
  ↓
FastAPI BackgroundTask 启动管线：
  1. 需求洞察 → 等待用户评分
  2. 问题建模 → 等待用户评分
  3. 专利分析 → 等待用户评分
  4. 方案生成 → 等待用户评分
  5. 方案评估 → 输出结果
  6. 完成 → 保存快照 → 成果转化面板
```

### 3.2 后台任务

```python
# services/workflow_service.py
class WorkflowService:
    """工作流编排服务。"""

    async def run_workflow(self, task_id: int):
        """启动工作流管线（后台执行）。"""
        sm = PhaseStateMachine()
        self._machines[task_id] = sm

        while sm.current != WorkflowPhase.COMPLETED:
            phase = sm.current
            result = await self._execute_phase(task_id, phase)

            # 等待用户评分（通过轮询通知前端）
            await self._wait_for_user_rating(task_id, phase)

            # 触发状态转移
            event = self._phase_to_done_event(phase)
            sm.transition(event)

        # 完成：保存快照
        await self._save_snapshot(task_id)
```

### 3.3 Phase 执行器

```python
async def _execute_phase(self, task_id: int, phase: WorkflowPhase):
    analyzers = {
        WorkflowPhase.DEMAND_PORTRAIT: DemandPortraitAnalyzer,
        WorkflowPhase.PROBLEM_MODELING: ProblemModelingAnalyzer,
        WorkflowPhase.PATENT_SEARCH: PatentSearchEngine,
        WorkflowPhase.SOLUTION_GEN: SolutionGenerator,
        WorkflowPhase.EVALUATION: SolutionEvaluator,
    }
    analyzer = analyzers[phase](self.ai_client)
    return await analyzer.analyze(...)
```

## 4. API 端点

```
POST /api/workflow/{task_id}/run
# 启动工作流（后台执行）
Response: { status: "started" }

GET /api/workflow/{task_id}/status
# 获取当前状态
Response: {
    phase: "problem_modeling",
    phaseStatus: {
        demand_portrait: "completed",
        problem_modeling: "running",
        patent_search: "pending",
        solution_gen: "pending",
        evaluation: "pending"
    },
    elapsed: 45.2
}

POST /api/workflow/{task_id}/rate
# 提交当前阶段用户评分
Body: { phase: string, ratings: [...] }

POST /api/workflow/{task_id}/proceed
# 确认进入下一阶段
```

## 5. 前端实现

### 5.1 三栏布局

工作流运行时，页面分为三栏：

```
┌─────────┬───────────────────────────┬─────────────────────┐
│ Sidebar │      Main Content         │  WorkflowPanel      │
│ (左栏)   │      (中栏)               │  (右栏)             │
│         │                           │                     │
│ 导航    │   ← 当前Agent步骤结果      │ ① 需求洞察Agent     │
│ 首页    │     按步骤更新展示         │    ✓ 已完成 12s     │
│ 知识库  │     - 需求画像结果         │ ② 问题建模Agent     │
│ 历史方案 │     - 问题建模分析         │    ● 运行中 ...     │
│ 专利检索 │     - 专利列表           │ ③ 专利分析Agent     │
│ 专利转化 │     - 方案卡片           │    ○ 等待中         │
│         │     - 评估雷达图           │ ④ 方案生成Agent     │
│         │     - 成果转化PDF          │    ○ 等待中         │
│ [管理员] │                           │ ⑤ 方案评估Agent     │
│         │                           │    ○ 等待中         │
│         │                           │ ⑥ 成果转化Agent     │
│         │                           │    ○ 等待中         │
└─────────┴───────────────────────────┴─────────────────────┘
```

- **WorkflowPanel** 为右侧专用面板，仅在工作流运行时（`isRunning === true`）显示
- 每个 Agent 步骤显示编号圆圈、名称、描述、状态标签（等待中/运行中/已完成/失败）
- 步骤完成后显示耗时（如 `已完成 12s`）
- 当前步骤完成后，中栏更新结果 UI，右栏进入下一步
