# 方案生成与评估

> 对应思维导图节点：方案生成（根据高相关的专利生成方案、用户交互相似度打分）、方案评估（目标需求、创新方向 → 生成方案的星星进行权重打分）

## 1. 方案生成

### 1.1 功能概述

```
方案生成
├── 根据高相关的专利生成方案
├── 结合需求画像和问题建模结果
└── 用户交互相似度打分（星星评分）
```

### 1.2 生成管线

```python
# algorithm/analyzers/solution_generator.py
class SolutionGenerator:
    """方案生成器 — 综合需求、建模和专利数据生成创新方案。"""

    async def generate(
        self,
        problem: str,
        demands: list[dict],          # 需求画像结果
        innovations: list[dict],       # 问题建模创新方向
        related_patents: list[dict],   # 相关专利
    ) -> list[dict]:

        # 1. 构建上下文
        context = self._build_context(demands, innovations, related_patents)

        # 2. 调用 AI 生成方案（多方案并行）
        solutions = await asyncio.gather(*[
            self._generate_one(problem, context, angle)
            for angle in ["技术", "成本", "效率", "用户体验"]
        ])

        # 3. 去重 + 评分排序
        ranked = self._rank_solutions(solutions)

        return ranked
```

## 2. 方案评估

### 2.1 功能概述

思维导图标注：

> 方案评估 → 目标需求、创新方向 → 生成方案的星星进行权重打分

方案评估基于四维度引擎：

```
方案评估
├── 创新性评估（专利相似度、技术进化度）
├── 可行性评估（约束推理、规则验证）
├── 完整性评估（推理链验证、交叉验证）
└── 成果转化评估（专利适用性、产业场景匹配）
```

### 2.2 四维评估引擎

```python
# algorithm/analyzers/evaluator.py
class SolutionEvaluator:
    """四维评估引擎。"""

    async def evaluate(
        self,
        solution: dict,
        demands: list[dict],
        innovations: list[dict],
        problem_context: str,
    ) -> dict:
        """对单个方案进行四维评估。"""

        innovation_score = await self._assess_innovation(solution, problem_context)
        feasibility_score = await self._assess_feasibility(solution, problem_context)
        completeness_score = await self._assess_completeness(solution, demands)
        transformation_score = await self._assess_transformation(solution)

        # 加权综合分（基于用户对需求和创新方向的评分）
        weights = self._calculate_weights(demands, innovations)
        overall = (
            innovation_score * weights["innovation"] +
            feasibility_score * weights["feasibility"] +
            completeness_score * weights["completeness"] +
            transformation_score * weights["transformation"]
        )

        return {
            "innovation": innovation_score,
            "feasibility": feasibility_score,
            "completeness": completeness_score,
            "transformation": transformation_score,
            "overall": overall,
            "details": {...}
        }
```

### 2.3 评估维度详情

| 维度 | 评估内容 | 权重依据 |
|------|---------|---------|
| 创新性 | 与现有专利的差异度、技术进化阶段 | 需求评分中的创新需求权重 |
| 可行性 | 技术约束、成本约束、时间约束 | 需求评分中的可行性需求权重 |
| 完整性 | 是否覆盖所有需求、逻辑链完整 | 需求覆盖率 |
| 成果转化 | 专利申请可行性、产业化潜力 | 创新方向中的转化需求权重 |

## 3. 数据库

```sql
CREATE TABLE evaluations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id INTEGER NOT NULL,
    solution_id INTEGER NOT NULL,
    innovation_score REAL,
    feasibility_score REAL,
    completeness_score REAL,
    transformation_score REAL,
    overall_score REAL,
    details JSON,
    created_at TEXT DEFAULT (datetime('now')),
    FOREIGN KEY (task_id) REFERENCES tasks(id),
    FOREIGN KEY (solution_id) REFERENCES solutions(id)
);
```
