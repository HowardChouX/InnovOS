# 方案生成与评估

> 对应思维导图节点：方案生成（根据高相关的专利生成方案、用户交互相似度打分）、方案评估（目标需求、创新方向 → 生成方案的星星进行权重打分）

## 1. 方案生成

### 1.1 功能概述

```
方案生成
├── 根据高相关的专利生成方案
├── 结合需求洞察和问题建模结果
└── 用户交互相似度打分（星星评分）
```

### 1.2 方案数据结构

```json
{
    "solutions": [
        {
            "id": "s1",
            "title": "方案标题",
            "description": "方案详细描述",
            "approach": "技术路线",
            "innovation_score": 0.85,
            "feasibility_score": 0.72,
            "based_patents": ["p1", "p3"],
            "based_innovations": ["in1", "in2"],
            "user_rating": null
        }
    ]
}
```

### 1.3 API 端点

```
POST /api/workflow/{task_id}/solutions/generate
Response: { solutions: [...] }

POST /api/workflow/{task_id}/solutions/rate
Body: { ratings: [{ solution_id: string, score: number }] }
```

## 2. 方案评估

### 2.1 功能概述

方案评估基于四维度引擎：

```
方案评估
├── 创新性评估（专利相似度、技术进化度）
├── 可行性评估（约束推理、规则验证）
├── 完整性评估（推理链验证、交叉验证）
└── 成果转化评估（专利适用性、产业场景匹配）
```

### 2.2 API 端点

```
POST /api/workflow/{task_id}/evaluate
Response: {
    evaluations: [
        {
            solution_id: "s1",
            innovation: 0.85,
            feasibility: 0.72,
            completeness: 0.90,
            transformation: 0.65,
            overall: 0.78
        }
    ]
}
```

## 3. 数据库

```sql
CREATE TABLE evaluations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id INTEGER NOT NULL,
    solution_id INTEGER NOT NULL,
    innovation_score REAL,
    feasibility_score REAL,
    completeness_score REAL,
    transformation_score REAL,
    overall_score REAL,
    details JSON,
    created_at TEXT DEFAULT (datetime('now')),
    FOREIGN KEY (task_id) REFERENCES tasks(id),
    FOREIGN KEY (solution_id) REFERENCES solutions(id)
);
```

## 5. 参考设计（RootSeek 模式）

方案生成模块可借鉴 RootSeek 的 `SolverCluster` 架构：

| RootSeek 特性 | InnovOS 可借鉴 |
|--------------|---------------|
| 8 Agent 并行（技术矛盾/物理矛盾/物-场/效应库/小人法/金鱼法/STC/裁剪） | 多角度并行方案生成 |
| `can_handle()` 准入检查 | 避免无效 Agent 执行 |
| bigram 文本去重 | 轻量级去重免向量依赖 |
| `SolutionConfidenceCalculator` 三维度置信度 | 方案可信度量化 |

评估模块可借鉴 RootSeek 的评估系统：

| RootSeek 特性 | InnovOS 可借鉴 |
|--------------|---------------|
| AI 评估（4 层检查） | 根因切断/矛盾解决/功能缺陷/IFR 距离 |
| 一致性门禁 G1-G4 | 防 AI 幻觉的代码级交叉验证 |
| 6 维度加权公式 | 参数达成/矛盾解决/根因切断/IFR-2 对齐/进化对齐/无新危害 |
