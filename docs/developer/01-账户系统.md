# 账户系统

> 对应思维导图节点：账户（类型、注册、登录、找回密码、账户作用）

## 1. 账户类型

| 类型 | 标识 | 权限范围 |
|------|------|---------|
| 管理员 | `role = 'admin'` | 全部功能 + 管理员后台 |
| 普通用户 | `role = 'user'` | 公共模块（首页、知识库、流程、历史、专利转化） |

管理员账户在系统初始化时通过 `seed.py` 自动创建。

## 2. 注册

### 2.1 当前实现（账号密码注册）

```
POST /api/auth/register
Body: { username: string, password: string }
```

- 用户名唯一性校验
- 密码经 bcrypt 哈希后存储
- 默认角色为 `user`

### 2.2 未来扩展（手机号注册）

思维导图中标注「手机号注册（以后扩展，当前只需要普通的注册就可以）」。

预留扩展方案：
- 在 `users` 表增加 `phone` 字段（可为空）
- 增加 SMS 验证码服务（阿里云/腾讯云 SMS API）
- 注册流程：发送验证码 → 验证 → 创建账户

## 3. 登录

### 3.1 账号密码登录（当前实现）

```
POST /api/auth/login
Body: { username: string, password: string }
Response: { token: string, user: { id, username, role } }
```

### 3.2 手机号登录（暂不开发）

思维导图中标注「手机号登录（暂时不开发）」。

## 4. 找回密码

### 4.1 手机号验证（规划中）

思维导图中标注「找回密码 → 手机号验证」。

当前实现：暂未开发。规划流程：
1. 用户输入注册手机号
2. 发送 6 位验证码（5 分钟有效）
3. 验证通过后设置新密码

### 4.2 临时方案

当前可通过管理员手动重置用户密码。

## 5. 账户作用

### 5.1 绑定个人数据实现数据隔离

所有业务数据（任务、知识库文档、通知、方案）均通过 `user_id` 字段关联当前用户，查询时强制过滤：

```python
# 所有查询都必须加 user_id 过滤
db.execute("SELECT * FROM tasks WHERE user_id = ?", (user["id"],))
```

前端通过 JWT Token 自动携带用户身份，后端通过 `get_current_user` 依赖注入解析。

### 5.2 管理员账户管理数据

管理员可：
- 查看所有用户的任务和数据（审计日志）
- 管理 API Key（添加/禁用/删除）
- 管理用户账户（查看/角色变更）
- 查看系统监控数据
- 管理专利数据库
- 向用户发送系统通知

## 6. 数据库表结构

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    role TEXT DEFAULT 'user',          -- 'admin' | 'user'
    phone TEXT,                        -- 预留：手机号
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
);
```
