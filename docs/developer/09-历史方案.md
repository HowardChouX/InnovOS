# 历史方案

> 对应思维导图节点：历史方案（用户保存的历次流程分析获得的整个数据流程保存）

## 1. 功能概述

历史方案模块保存用户每次完整流程分析的所有数据，允许用户回溯、对比和复用历史分析结果。

```
历史方案
├── 列表浏览（按时间排序）
├── 详情查看（完整流程数据回放）
├── 数据导出
└── 复用为新任务
```

## 2. 数据存储

### 2.1 存储策略

每次工作流完成后，将完整流程数据快照保存到 `workflows` 表的 `snapshot` 字段：

```json
{
    "task_id": 42,
    "problem": "如何提高散热效率",
    "completed_at": "2024-01-15T10:30:00Z",
    "snapshot": {
        "demand_portrait": {
            "demands": [...],
            "user_ratings": [...]
        },
        "problem_modeling": {
            "function_analysis": {...},
            "causal_chain": {...},
            "contradictions": [...],
            "innovations": [...],
            "user_ratings": [...]
        },
        "patent_search": {
            "results": [...],
            "user_ratings": [...]
        },
        "solution_gen": {
            "solutions": [...],
            "user_ratings": [...]
        },
        "evaluation": {
            "evaluations": [...]
        }
    }
}
```

### 2.2 数据库表

```sql
-- workflows 表增加 snapshot 字段
ALTER TABLE workflows ADD COLUMN snapshot JSON;
ALTER TABLE workflows ADD COLUMN completed_at TEXT;
```

## 3. API 端点

```
GET /api/history
# 获取历史方案列表
Response: {
    data: [{ task_id, problem, completed_at, summary }],
    total: number
}

GET /api/history/{task_id}
# 获取历史方案详情（完整流程数据）
Response: {
    data: { task_id, problem, snapshot: {...} }
}

POST /api/history/{task_id}/reuse
# 基于历史方案创建新任务
Response: { new_task_id: number }

DELETE /api/history/{task_id}
# 删除历史方案
```
