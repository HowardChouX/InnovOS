# 需求洞察

> 对应思维导图节点：需求洞察（知识图谱信息分析采用 CherryStudio 接入方式、七来源分析、IFR 理想解、九屏幕法、金鱼法、STC 算法、目标需求 → 需求列表 → 用户交互评分）

## 1. 功能概述

需求洞察阶段通过多种 TRIZ 分析工具对用户输入的问题进行多维度分析，生成需求列表，供用户在前端进行交互式评分。

```
需求洞察
├── 知识库 RAG 检索（CherryStudio 方式）→ 上下文增强
├── 七来源分析（RootSeek）→ 创新来源
├── IFR 理想解（RootSeek）→ 理想最终结果
├── 九屏幕法（RootSeek）→ 多尺度分析
├── 金鱼法（RootSeek）→ 隐性需求挖掘
├── STC 算法（RootSeek）→ 参数极端化分析
└── 目标需求 → Agent 思考结果列出可能需求列表
    → 用户前端交互评分（星星评分）
```

## 2. 架构设计

### 2.1 分析器管线

所有分析器继承自 RootSeek 的 `AIAnalyzer` 基类：

```python
# algorithm/analyzers/base.py（从 RootSeek 移植）
class AIAnalyzer:
    def __init__(self, ai: AIBase):
        self.ai = ai

    async def call_ai(self, system_prompt, user_prompt, ...) -> dict | str | None:
        return await self.ai.call_ai(...)
```

### 2.2 需求洞察编排器

```python
# algorithm/analyzers/demand_portrait.py
class DemandPortraitAnalyzer:
    """需求洞察编排器 — 并行运行多种分析工具，汇总需求列表。"""

    def __init__(self, ai_client):
        self.resource_analyzer = ResourceAnalyzer(ai_client)
        self.ifr_generator = IFRGenerator(ai_client)
        self.nine_screens = NineScreensAnalyzer(ai_client)
        self.goldfish = GoldfishAnalyzer(ai_client)
        self.stc = STCAnalyzer(ai_client)

    async def analyze(self, problem: str, knowledge_context: str = "") -> dict:
        """并行运行所有分析工具，生成需求列表。"""

        # 并行执行所有分析
        results = await asyncio.gather(
            self.resource_analyzer.analyze(problem),
            self.ifr_generator.analyze(problem),
            self.nine_screens.analyze(problem),
            self.goldfish.analyze(problem),
            self.stc.analyze(problem),
        )

        # 汇总需求列表
        demands = self._aggregate_demands(results)

        return {
            "resource_analysis": results[0],
            "ideal_final_result": results[1],
            "nine_screens": results[2],
            "goldfish": results[3],
            "stc": results[4],
            "demands": demands,  # 统一需求列表
        }
```

## 3. 各分析器详情（从 RootSeek 迁移）

### 3.1 七来源分析（ResourceAnalyzer）

**RootSeek 源文件**：`src/ai/analyzers/resource_analyzer.py`

分析系统中可用的物质、场、信息、功能、时间、空间、结构七种资源。

### 3.2 IFR 理想解（IFRGenerator）

**RootSeek 源文件**：`src/ai/analyzers/ifr_generator.py`

生成理想最终结果（Ideal Final Result），定义系统在理想状态下的行为。

### 3.3 九屏幕法（NineScreensAnalyzer）

**RootSeek 源文件**：`src/ai/prompts/nine_screens.py`

从时间维度（过去/现在/未来）× 系统维度（子系统/系统/超系统）的 9 个视角分析问题。

### 3.4 金鱼法（GoldfishAnalyzer）

**RootSeek 源文件**：`src/ai/prompts/goldfish.py`

通过"金鱼许愿"的方式，引导用户表达隐性需求和真实目标。

### 3.5 STC 算法（STCAnalyzer）

**RootSeek 源文件**：`src/ai/prompts/stc_operator.py`

Size（尺寸）、Time（时间）、Cost（成本）三个参数的极端化思考，突破思维定式。

## 4. 需求列表与用户评分

### 4.1 需求列表数据结构

```python
{
    "demands": [
        {
            "id": "d1",
            "source": "七来源分析",       # 来源工具
            "category": "功能需求",        # 需求类型
            "description": "提高散热效率",  # 需求描述
            "priority": 0,                # 初始优先级
            "user_rating": None           # 用户评分（1-5 星）
        },
        ...
    ]
}
```

### 4.2 前端交互评分

```typescript
// features/workflow/DemandPortrait/DemandPortraitPanel.tsx
function DemandRating({ demands, onRate }) {
  return (
    <div>
      {demands.map(demand => (
        <DemandCard key={demand.id}>
          <span className="source">{demand.source}</span>
          <p>{demand.description}</p>
          <StarRating
            value={demand.user_rating}
            onChange={(rating) => onRate(demand.id, rating)}
          />
        </DemandCard>
      ))}
    </div>
  );
}
```

### 4.3 评分提交

```
POST /api/workflow/{task_id}/demand-rate
Body: { ratings: [{ demand_id: string, score: number }] }
```

## 5. 状态机阶段

```python
# algorithm/workflow_state_machine.py
class WorkflowPhase(Enum):
    DEMAND_PORTRAIT = "demand_portrait"       # 需求画像
    PROBLEM_MODELING = "problem_modeling"     # 问题建模
    PATENT_SEARCH = "patent_search"           # 专利检索
    SOLUTION_GEN = "solution_gen"             # 方案生成
    EVALUATION = "evaluation"                 # 方案评估
    COMPLETED = "completed"                   # 完成
```

需求画像是第一阶段，完成后通过 `DEMAND_PORTRAIT_DONE` 事件转入问题建模。

## 6. 迁移适配要点

从 RootSeek 迁移时需注意：

| 差异项 | RootSeek | InnovOS |
|--------|----------|---------|
| AI 客户端 | `AsyncOpenAI` 直接实例化 | 通过 `key_manager` 获取 Key |
| 提示词 | `ai/prompts/` 独立文件 | `algorithm/prompts/demand_portrait/` |
| 调用方式 | `self.ai.call_ai()` | 相同（保持 AIAnalyzer 接口） |
| 并行控制 | `MAX_PARALLEL_PAIRS = 10` | 按需配置 |
